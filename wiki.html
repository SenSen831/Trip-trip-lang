What is Java?

Java is a high-level, class-based, object-oriented programming language designed to have as few implementation dependencies as possible. "Write Once, Run Anywhere" (WORA) — compiled Java code runs on all platforms that support Java without recompilation.

Your First Program

Java
// HelloWorld.java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Data Types

Java has two categories: primitive (int, double, boolean, char…) and reference types (objects, arrays, strings).

Java
int age = 21;
double gpa = 3.85;
String name = "Alice";
boolean isStudent = true;
System.out.println(name + " | GPA: " + gpa);

Control Flow

Java
int score = 85;
if (score >= 90) {
    System.out.println("A");
} else if (score >= 80) {
    System.out.println("B");
} else {
    System.out.println("C");
}
for (int i = 1; i <= 3; i++) {
    System.out.println("Loop: " + i);
}
💡 Tip

Java is case-sensitive. Main and main are different identifiers — the entry point must be public static void main.

Inheritance

Java
class Animal {
    String name;
    void speak() { System.out.println(name + " makes a sound"); }
}
class Dog extends Animal {
    Dog(String n) { name = n; }
    @Override
    void speak() { System.out.println(name + " says: Woof!"); }
}
Dog d = new Dog("Rex");
d.speak();
💡 Tip

Use private fields with public getters/setters (encapsulation) to protect your object's internal state from unintended modification.

Exception Handling
Java🟡 Intermediate

Try / Catch / Finally

Exceptions are runtime errors. Java uses try-catch-finally to handle them gracefully without crashing the program.

Java
try {
    int result = 10 / 0;
    System.out.println(result);
} catch (ArithmeticException e) {
    System.out.println("Caught: " + e.getMessage());
} finally {
    System.out.println("Finally always runs");
}

Custom Exceptions

Java
class AgeException extends Exception {
    AgeException(String msg) { super(msg); }
}
static void checkAge(int age) throws AgeException {
    if (age < 18) throw new AgeException("Too young!");
    System.out.println("Age accepted: " + age);
}
try { checkAge(15); } catch (AgeException e) { System.out.println(e.getMessage()); }
try { checkAge(21); } catch (AgeException e) { System.out.println(e.getMessage()); }
💡 Tip

Always catch the most specific exception first. Never catch Exception or Throwable unless you have a very good reason.

Async & Promises
JavaScript🔴 Advanced

Promises

JavaScript
const fetchData = new Promise((resolve, reject) => {
    setTimeout(() => resolve("✅ Data loaded!"), 500);
});
fetchData
    .then(data => console.log(data))
    .catch(err => console.error(err))
    .finally(() => console.log("Done"));

Async / Await

JavaScript
async function getUser() {
    try {
        const result = await new Promise(res =>
            setTimeout(() => res({ name: "Alice", role: "student" }), 300)
        );
        console.log("User:", result.name, "| Role:", result.role);
    } catch(e) {
        console.error(e);
    }
}
getUser();
💡 Tip

Always wrap await calls in try/catch — rejected promises that aren't caught cause unhandled rejection errors.

Semantic HTML

Why Semantic HTML?

Semantic elements clearly describe their meaning to the browser and developer — improving SEO, accessibility, and maintainability.

HTML
<!-- Non-semantic -->
<div class="header">...</div>
<!-- Semantic (preferred) -->
<header>Site Logo & Nav</header>
<nav>Navigation links</nav>
<main>
  <article>Blog post content</article>
  <aside>Related links</aside>
</main>
<footer>Copyright info</footer>

Keyframe Animations

CSS
@keyframes pulse {
  0%   { transform: scale(1); opacity: 1; }
  50%  { transform: scale(1.1); opacity: 0.7; }
  100% { transform: scale(1); opacity: 1; }
}
.badge {
  animation: pulse 2s infinite ease-in-out;
}
@keyframes slideIn {
  from { transform: translateX(-100%); opacity: 0; }
  to   { transform: translateX(0); opacity: 1; }
}
💡 Performance

Animate only transform and opacity for smooth 60fps animations. Animating width, height, or top causes layout recalculation and is much slower.
